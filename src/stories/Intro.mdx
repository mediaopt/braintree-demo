# Braintree Payment Demo

This demo is intended to showcase the Braintree payment methods integration with commercetools. It covers the checkout session implementation and flows for different payment methods, as well as some of the project/shop level settings supported by the Braintree commercetools connector.

## Setup

Our own demo environment is already provided. To test it with your own commercetools project, you can clone the repository and set up your own environment variables, install the Braintree commercetools checkout connector for your project and create checkout application.

We strongly recommend to use for this demo the API client with commercetools checkout permissions only, to avoid any unwanted side effects on your commercetools project.

We use Vite for the development environment, so the environment variables should be prefixed with `VITE_` to be available in the code. If your shop is built with a different technology, you can adjust the environment variable names.

## Structure

We have two main demo modes in the project: checkout only loads the commercetools checkout module with predefined product and cart settings. Playground allows to cover settings that should be dynamically obtained from server - like shipping modes available for different countries and preview the cart summary before starting the checkout process.

We only provide cart creation in order to demonstrate the full flow from product selection to checkout, therefore it contains some hardcoded values and is not intended to be used as a reference for your actual cart implementation. The cart is created with a default customer, and the product selection is limited to a few hardcoded product IDs. Please use official commercetools documentation and SDKs for the actual cart implementation in your shop.

The real implementation of the checkout session is covered by the `CheckoutLoader` component, which is used in both modes.

### Checkout modes

- Standard: commercetools standard checkout flow, must be called from a checkout page.
- Express: commercetools express checkout flow, can be called for any page. We recommend it for buying the product directly from the product page.

See also commercetools documentation for [checkout](https://docs.commercetools.com/checkout/connectors-and-applications).

## Storybook

The demo is built with Storybook, which allows to easily test different scenarios and configurations. All payments supported by the Braintree connector are available in this Storybook, and you can also adjust the cart settings to see how they affect the checkout flow. We have two main demo modes - Chekout and playgorund.

Checkout mode gives the end-to-end checkout experience starting from the moment the customer clicks the checkout button and until the payment is completed. It utilized minimalistic cart configuration with storybook controls. Playground gives more visual experience of the cart configuration and covers the settings, that should be loaded dynamically, such as shipping for different countries.

Both modes are configured to demonstrate the following checkout flows:

### Flows

A valid commercetools cart is required to start the checkout session, but different flows have different additional requirements. The default checkout mode settings will automatically satisfy the requirements for all flows.

#### Standard (Checkout)

**_Cart requirements: cart total price above 0, shipping address, billing address and customer email_**

The default mode. You can add products, adjust cart-level settings (country, tax mode, shipping, discounts), and then click **Load checkout** to start the Braintree payment session. In the Playground mode a cart is created on mount and can be edited later, in checkout mode - upon button press the cart is created with all predefined settings.

#### Buy Now

**_Cart requirements: cart total price above 0, cart has country and currency set_**

Dedicated to buy a product directly from product page. In checkout mode only one product can be selected for this mode. In playground mode each product card has a **Buy Now** button — clicking it creates the cart with that product and immediately loads the checkout session.

#### Vault without purchase

This flow is not currently supported by the Braintree connector. If you need it, please request support in the [main connector repository](https://github.com/mediaopt/braintree-commercetools-connector).

### Demo Cart

In our demo we include both standard shop features, such as product selection and shipping, and some of the more advanced features that are usually set on at least shop level or even project level. We simplify the things not directly relevant for payment process, such as setting the country also sets a random address within this country.

- Products: we use for the demo the products with some distinctive features, namely:
  - Standard product, which is has no custom features and serves as a baseline for testing the checkout flow.
  - Free Item: features commercetools giftLineItem (buy at least 1, get exactly 1 free). See [commercetools documentation](https://docs.commercetools.com/api/projects/cartDiscounts#cartdiscountvaluegiftlineitem) for more details.
  - Exact Discount: features product discount with a fixed amount.
  - Percent Discount: features discount on the whole cart with a percentage amount if this product is included.

See also commercetools documentation for [product](https://docs.commercetools.com/api/projects/productDiscounts) and [cart](https://docs.commercetools.com/api/projects/cartDiscounts) discounts.

The price shown next to each product in the Playground is taken from the default **master variant** of that product. Commercetools supports country-specific and channel-specific prices, so the actual amount charged during checkout may differ once a billing country is set, please see a cart summary for an actual price.

- Country: sets the country for the cart, which can affect the prices and shipping options. As this is a demo, we support limited amount of countries and shipping options. Also, it automatically sets the billing and shipping addresses with a corresponding country.
- Signed in customer: assigns a customer to the cart. If the connector has vaulting enabled and customer is signed in, Braintree saved payment methods will be loaded automatically.

Other settings directly correspond to the [commercetools cart properties](https://docs.commercetools.com/api/projects/carts).
